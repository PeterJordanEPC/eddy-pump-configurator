# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use this repository's **Security** tab to submit a private vulnerability report to the maintainers.

Do not include customer information, credentials, tokens, production database values, or HubSpot data in a report. Include only the minimum reproduction details needed to investigate safely.

## Supported version

Only the currently deployed `main` branch is supported. Security fixes are tested through CI and released through the protected pull-request workflow.
