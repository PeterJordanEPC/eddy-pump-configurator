# EDDY Pump — Pump & Dredge Configurator

Production-hardened static React configurator hosted on GitHub Pages. Customer intake is sent over HTTPS to the private Railway API; the public repository contains no credentials, CRM tokens, customer records, quote pricing, or approval state.

## Live site

https://configurator.eddypump.com/

HTTP is redirected to HTTPS by GitHub Pages.

## Architecture

```text
configurator.eddypump.com (GitHub Pages)
        |
        | HTTPS, validated JSON, explicit submission + idempotency
        v
Railway private API + PostgreSQL
```

The browser recommendation is preliminary. The backend validates the answers, recomputes the authoritative product family, stores encrypted customer information, and marks engineering-review requirements.

HubSpot quote creation, customer email, and follow-ups remain disabled until separate launch approval and end-to-end verification.

## Development

Requirements: Node.js and npm.

```bash
npm install
npm run check
```

- `src/app.jsx` — editable React source
- `app.js` — committed minified production bundle generated by esbuild
- `scripts/check-build.mjs` — deterministic production-hardening checks
- `images/eddy-pump-corporation-logo.webp` — optimized local corporate logo

Do not edit `app.js` directly. Run `npm run build` after changing `src/app.jsx`.

## Configurator flow

- Dredging: material → cubic yards/hour → deployment → power
- Process pump: material → GPM range → discharge head when applicable → configuration → power
- GPM ranges map to 1, 2, 3, 4, 6, 8, 10, 12, or 16-inch pump classes; the tool never recommends a 5-inch class
- Head is captured for engineering review and never overrides the flow-sized pump, selected configuration, or power
- Project notes are always visible in the main contact section; detailed engineering fields use progressive disclosure

## Product photography

Cards use a local image when a corresponding file is available and otherwise fall back to lightweight blueprint artwork. Approved product images can be placed in `images/` with these names:

| Filename | Use |
|---|---|
| `dredging.jpg` | Dredging application |
| `slurry.jpg` | Process pump application |
| `sand.jpg`, `sludge.jpg`, `tailings.jpg`, `debris.jpg`, `other.jpg` | Materials |
| `flow.jpg` | Flow, production, and head ranges |
| `electric.jpg`, `hydraulic.jpg`, `diesel.jpg` | Power |
| `excavator.jpg`, `cable.jpg`, `remote.jpg`, `sled.jpg`, `diver.jpg`, `auger.jpg` | Dredging deployment |
| `flooded.jpg`, `submersible.jpg`, `selfpriming.jpg` | Process-pump configuration |

Use approved landscape assets around 1200 pixels wide with consistent lighting and composition. Do not hotlink production photos from third-party sites.

## Security controls

- HTTPS-only intake guard
- Restrictive Content Security Policy
- Exact-version React scripts with Subresource Integrity
- Server-side schema validation and authoritative recommendation
- Explicit submit action with native required-field validation and no automatic marketing subscription
- Honeypot and backend rate limiting
- Idempotency keys to prevent duplicate intake
- Encrypted PII at rest in PostgreSQL
- Restricted CORS
- No direct browser-to-HubSpot integration
- Approval gates for recommendations and eventual customer sends

The temporary Railway-generated API hostname remains supported until the custom API DNS record and certificate are fully validated.
